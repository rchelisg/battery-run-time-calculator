## Git Workflow — ALWAYS FOLLOW THESE STEPS
Before starting work:   git pull
After finishing work:   git add . && git commit -m "description" && git push

- Never leave uncommitted changes when ending a session
- Always pull before starting work on any computer
- Write clear, descriptive commit messages explaining what changed

## Coding Standards
- Vanilla HTML, CSS, JavaScript only — no frameworks
- Simple, well-commented code
- Use clear, descriptive variable and function names
- Keep functions small and single-purpose
- Comment any logic that is not immediately obvious
